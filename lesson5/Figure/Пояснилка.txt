Я пытался выполнить дополнительное задание с реализацией дефолтных методов интерфейса.
Изначально методы для вычисления периметров и площадей фигур были без реализации.

public interface Figure {

  double perimeter();
  double area();

}

Чтобы решить задачу с реализацией методов в самом интерфейсе, на ум пришло создание const переменных для методов perimeter(); и area();
И вложение в них простой формулы для вычисления периметра и площади для прямоугольников.
А уже в остальных классах других фигур, переопределял этот метод для треугольников и кругов.
Таким образом метод получился заранее реализованный внутри интерфейса для прямоугольных фигур.
